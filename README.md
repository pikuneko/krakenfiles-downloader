# 🚀 KrakenFiles 自動ダウンロード＆RAR/ZIP自動解凍システム

ユーザーが入力した複数の KrakenFiles の URL から、広告やポップアップの妨害を物理的に排除しつつ自動でファイルをダウンロードし、完了後に一括で RAR/ZIP ファイルを解凍する Python スクリプトです。

---

## 🛠️ 機能特徴

1. **ゾンビ連打＆広告強制クローズ機能**
   - クリック時に仕込まれた悪質な広告用別タブの自動検知・強制閉鎖。
   - ページ全体が広告に書き換えられた場合の自動「ページ戻り（Back）」と再試行ループ。
2. **ダウンロード完了シグナル検知**
   - バックグラウンド（`chrome://downloads`）を自動監視し、保存が 100% 終わるまで次の URL へ進まない安全設計。
3. **RAR/ZIP 自動一括解凍**
   - 全ダウンロード終了後、PC 内の解凍ツール（7-Zip または WinRAR）を自動検出。
   - ファイル名と同名のフォルダを自動生成して綺麗に一括解凍。

---

## 📦 前提環境と準備

当システムを動かすには以下の環境が必要です。

* **OS**: Windows 10 / 11
* **Python**: バージョン 3.10 以上
* **Google Chrome**: 実装されている最新のブラウザ

### ⚡ クイック環境構築
フォルダ内にある **`setup.bat`** をダブルクリックして実行してください。
自動的に管理者権限へ昇格（BOM付きUTF-8対応）し、以下の作業をワンクリックで一括代行します。
- `pip` の最新化
- 必要なライブラリ（`undetected-chromedriver`, `beautifulsoup4`, `requests`）の導入
- パソコン内に 7-Zip が無い場合、公式から最新版をバックグラウンド自動インストール

---

## 🚀 使い方・運用手順

### 1. 起動前のルール
- 普段お使いの通常の Google Chrome のウィンドウは、**すべて閉じた状態**で実行することをおすすめします。

### 2. プログラムの実行
VS Codeのターミナル、またはコマンドプロンプトで以下のコマンドを実行します。

```bash
python main.py
```

### 3. URL の入力
1. 画面に `url1:` と表示されるので、KrakenFiles の URL を貼り付けて `Enter` を押します。
2. `url2:`, `url3:` と続けて、ダウンロードしたい数だけまとめて入力します。
3. **入力を終了する場合は、何も入力せずにそのまま `Enter` キーを押します。**

## 🛑 徹底網羅：トラブルシューティング（Q1〜Q21）

### 【ブラウザ・システム起動関連】

#### 🟢 Q1. 「session not created: This version of ChromeDriver only supports...」という赤いエラーが出る
- **原因**: ツール（`undetected-chromedriver`）が認識しているバージョンと、現在あなたのPCに入っている実際の Google Chrome のバージョンがズレています。
- **対策**: **`Apps` フォルダ内にある `Q1.bat` をダブルクリックして実行してください。** あなたのPCの現在のChromeのバージョンを自動解析し、`main.py` のバージョン指定コードを数秒で自動修正します。手動でコードを書き換える必要はありません。修正が完了したら再度 `main.py` を実行してください。

#### Q2. 画面が起動した直後（newtab）のままフリーズして動かない
- **原因**: 既存の重いChromeプロファイルや、ログインデータの同期競合によりブラウザの通信が内部ロック（フリーズ）しています。
- **対策**: 本スクリプトはプロファイル依存を無くし、クリーンな状態で起動する仕様に最適化されています。必ずお使いの通常のブラウザ（Profile 9など）のウィンドウを完全に閉じてからスクリプトを実行してください。

#### Q3. `setup.bat` を実行すると文字化けして画面がすぐに閉じてしまう
- **原因**: バッチファイルがShift-JISなどで保存されており、Windowsが日本語を正しく認識できていません。
- **対策**: `setup.bat` のエンコードを必ず「UTF-8（BOM付き）」にして保存してください。冒頭の `chcp 65001` が文字コードをUTF-8に強制固定するため、文字化けを防ぐことができます。

#### Q4. バッチファイル実行時に「管理者権限への昇格」画面がループして起動しない
- **原因**: Windowsのユーザーアカウント制御（UAC）が最高レベルに設定されているか、セキュリティソフトにブロックされています。
- **対策**: バッチファイルを直接右クリックし、「管理者として実行」を明示的に選択して起動してください。

#### Q5. プログラム終了時にコンソールに「[WinError 6] ハンドルが無効です」と出力される
- **原因**: プログラム終了時の片付け（`driver.quit()`）の際、すでに無効になったプロセスを操作しようとして発生した一時的な警告です。
- **対策**: 内部の `try-except` により安全に受け流す処理を組んでいるため、ダウンロード処理やファイルの解凍自体への悪影響は一切ありません。無視して問題ありません。

#### 🟢 Q18. パソコンに Google Chrome 自体がインストールされていない
- **原因**: 実行環境に必須ブラウザが欠落している場合。
- **対策**: **`Apps` フォルダ内にある `Q18.bat` をダブルクリックして実行してください。** 自動的に管理者権限へ昇格し、Google公式サーバーから最新安定版のChrome（64bit）のスタンドアロンインストーラーをバックグラウンドで安全に取得・サイレントインストールします。

### 【広告ブロック・ボタン誤動作関連】

#### Q6. ボタンを押した直後にアークナイツなどの広告ページに飛ばされてしまう
- **原因**: ボタン自体に1回目のクリック時だけ広告へ強制転送するJavaScript（インラインイベント）が仕込まれています。
- **対策**: プログラムが「別タブが開いたら即座に強制閉鎖（`.close()`）」、「元のタブが書き換えられたら即座にブラウザのバック（`driver.back()`）」を自動で最大8回繰り返す『ゾンビ連打システム』を搭載しています。基本的には自動で引き戻されますが、もしループから抜け出せない場合は、ブラウザ画面の「←（戻る）」を手動で1回押して補助してください。

#### Q7. 画面上に「PopUP（ポップアップ）がブロックされました」と表示されて保存が始まらない
- **原因**: 高速なクリック命令により、ブラウザのセキュリティが「悪質な不正ポップアップ」と誤認識してダウンロードを遮断しています。
- **対策**: スクリプトの起動オプションにて、Chrome内部の `popups` と `automatic_downloads` 設定を裏側から常に「1（許可）」に強制書き換えしています。また、JavaScriptではなく物理的なクリックイベント（`MouseEvent`）をシミュレートすることで、ブラウザのガードを無効化しています。

#### Q8. サーバー応答で「"msg":"captcha not valid" (500エラー)」が出て引っこ抜けない
- **原因**: 画面を開かない純粋な通信（Requests等）で直接データを要求したため、Cloudflare Turnstile（パズル認証）のセキュリティトークンが無い不正なBotアクセスとして弾かれています。
- **対策**: 当システムはあえて画面操作（Selenium）に戻すことで、ブラウザ上で合法的にパズルをクリアします。画面にCloudflareの「人間であることを証明してください」が出現した場合は、**最初の12秒の待機時間中に対象のチェックボックスを手動でクリック**して突破してください。その後の連打や保存はすべて自動化されます。

#### Q9. ページ内の広告要素を消し去るJavaScriptはどうやって動いているの？
- **原因**: 手動で配置したAdGuardフォルダがChromeの仕様変更で弾かれるリスクに備えた、プログラムによる直接お掃除機能です。
- **対策**: ボタンをクリックする直前に、画面上のすべての要素のイベントリスナー（広告の罠）を解除したクローン要素に置き換え、さらにCSSで高階層（`zIndex > 10`）に配置された透明な膜（オーバーレイdivやiframe）をプログラムが自動で `.remove()` して更地にする仕組みになっています。

#### Q10. 広告ブロック用のAdGuard DNS（94.140.14.14）を元に戻したい
- **原因**: パソコン全体に適用した広告ブロック設定をリセットしたい場合。
- **対策**: Windowsの「ネットワークとインターネット」の設定から変更した「DNSサーバー割り当て」の編集画面をもう一度開き、設定を「手動」から「自動 (DHCP)」に戻して保存してください。

### 【ダウンロード・完了検知関連】

#### Q11. ダウンロード完了シグナルの検知画面（chrome://downloads/）が開いたままになる
- **原因**: プログラムがバックグラウンドでダウンロードの状態をリアルタイム監視するために、一時的に新規タブとして開いています。
- **対策**: 監視ループ内で「ダウンロード中（`IN_PROGRESS`）」の状態が終了したのを検知すると、プログラムが自動的にそのタブを `.close()` して元のページに引き返すように設計されています。手動で閉じる必要はありません。

#### Q12. ダウンロード中にファイルが途切れて、次のURLへ進んでしまう
- **原因**: 前のページの読み込み判定が甘く、ダウンロードが完全に開始される前に次へ進んでしまう現象です。
- **対策**: ループチェック（`wait_for_download_complete`）の内部で、Chrome一時ファイルである `.crdownload` がローカルフォルダ内から完全に消滅するまで次のURLへの遷移をがっちりホールドする安全な物理ダブルチェック構造を導入しています。

#### 🟢 Q13. 5分以上かかる超特大ファイルをダウンロードすると途中でタイムアウトする
- **原因**: プログラム内の安全装置（標準値: `timeout=300`）により、5分経過すると自動で次に進む仕様になっているためです。
- **対策**: **`Apps` フォルダ内にある `Q13.bat` をダブルクリックして実行してください。** 画面の指示に従って「何分待ちたいか」を半角数字（分）で入力するだけで、`main.py` 内の制限秒数を自動計算して一瞬で上書き修正します。手動でコードを書き換える必要はありません。

#### Q14. ファイルの保存先（保存フォルダ）をデスクトップ等に変更したい
- **原因**: 標準状態では、Windowsが管理するユーザーの「ダウンロード（Downloads）」フォルダに格納されます。
- **対策**: 保存先はGoogle Chrome本体の設定に依存します。変更したい場合は、一度通常のChromeを開き、「設定」➔「ダウンロード`」➔「位置」から保存先を任意のフォルダに変更してください。自動起動ブラウザにもその設定が反映されます。

#### 🟢 Q22. 広告転送（ゾンビシステム）のループ回数を変更したい
*   **原因・状況**: KrakenFiles側の広告配置が激化し、標準の8回リトライ（別タブ閉鎖・自タブ引き戻し）では突破しきれずに処理が止まってしまう、または逆に回数を減らして早く諦めさせたい場合。
*   **解決方法**: `Apps/Q22.bat` を実行してください。対話型画面が表示されますので、新しく設定したい試行回数（例: `12` など）を入力してエンターを押すだけで、`main.py` の内部パラメーターが自動的に書き換わります。

### 【RAR/ZIP自動解凍関連】

#### Q15. ダウンロードが終わったのに圧縮ファイルの解凍工程に進まない
- **原因**: パソコンに「7-Zip」または「WinRAR」が標準のインストール場所（`C:\Program Files`）に配置されていません。
- **対策**: `setup.bat` をダブルクリックして実行し、公式の7-Zipを正しいディレクトリへ自動導入させてください。解凍システムが自動検知し、ファイルと同名のフォルダへ綺麗に一括解凍されるようになります。

#### Q16. 7-ZipとWinRARの両方がPCに入っている場合、どちらが優先される？
- **原因**: 解凍関数の優先順位。
- **対策**: プログラム内部の設計上、より軽量でコマンドラインの安定性が高い「7-Zip」を最優先で検出するように組んでいます。7-Zipが無い場合にのみ、自動的に「WinRAR」を呼び出すロジックに切り替わります。

#### 🟢 Q17. 解凍に成功したら元の圧縮（.rar/.zip）ファイルを自動でゴミ箱（消去）に送りたい
- **原因**: 解凍後の重複ファイルを自動で片付けたい場合。
- **対策**: **`Apps` フォルダ内にある `Q17.bat` をダブルクリックして実行してください。** 画面の指示に従って、解凍完了後に元のRAR/ZIPファイルを自動消去するか（Y/N）を対話形式で選択するだけで、`main.py` の内部コードを自動的に書き換えて切り替えます。

#### 🟢 Q21. 自動解凍の対象形式にZIPファイルも含めたい / 戻したい
- **原因**: ダウンロードされる拡張子がRARだけでなく、ZIP形式も混ざっている場合。
- **対策**: **`Apps` フォルダ内にある `Q21.bat` をダブルクリックして実行してください。** 画面の指示に従って、自動解凍の対象にZIPファイルも含めるか（Y/N）を対話形式で自由に切り替えることができます。
